# Unit 1: awsomeshop-frontend — 技术栈决策

---

## 1. 核心技术栈

### 前端框架
- **决策**: React 19 + TypeScript
- **说明**: 使用最新版本的 React 和 TypeScript 开发
- **特性**:
  - 类型安全的开发体验
  - 现代化的 Hooks API
  - 强大的生态系统

### UI 组件库
- **决策**: Material UI (MUI) v6
- **依赖**:
  - @emotion/react
  - @emotion/styled
- **说明**: 成熟的 React UI 组件库，提供丰富的组件和主题定制能力

### 状态管理
- **决策**: Zustand v5
- **说明**: 轻量级的状态管理库，API 简洁，性能优秀

### 路由方案
- **决策**: React Router v7
- **说明**: React 生态中的标准路由解决方案

---

## 2. 浏览器兼容性

### 目标浏览器
- **决策**: 现代浏览器最新 2 个主版本
- **支持列表**:
  - Chrome 最新 2 版本
  - Firefox 最新 2 版本
  - Safari 最新 2 版本
  - Edge 最新 2 版本
- **不支持**: IE11
- **理由**: 内部系统，可控制用户浏览器环境

### Browserslist 配置
```
last 2 Chrome versions
last 2 Firefox versions
last 2 Safari versions
last 2 Edge versions
not IE 11
```

---

## 3. 性能策略

### 首屏性能目标
- **FCP 目标**: P95 ≤ 2 秒
- **测量环境**: 4G 网络，桌面端 Chrome

### 构建工具
- **决策**: Vite 7
- **说明**: 现代化的前端构建工具，快速的热更新和优化的生产构建

### 代码分割
- **决策**: 基于 Vite 和 React Router 的路由级代码分割
- **策略**: 路由级代码分割，按需加载页面组件

### 包体积限制
- **决策**: 通过 Vite 的构建优化和 Tree Shaking 控制包体积
- **说明**: Vite 会自动进行代码分割和优化

---

## 4. 可访问性策略

### 合规标准
- **决策**: WCAG 2.1 AA 级别
- **理由**: 企业内部系统应满足基本可访问性要求

### 实现要点
| 要求 | 实现方式 |
|------|---------|
| 键盘导航 | 所有交互元素可 Tab 聚焦 |
| 语义化 HTML | 使用正确的 HTML 标签 |
| 颜色对比度 | 正文 4.5:1，大文本 3:1 |
| 焦点可见 | 自定义焦点样式 |
| 屏幕阅读器 | aria-label、aria-live |

---

## 5. 安全策略

### XSS 防护
- **决策**: 依赖 React 的自动转义机制
- **说明**: React 默认会对所有渲染内容进行转义
- **补充要求**:
  - 禁止使用 dangerouslySetInnerHTML 插入用户内容
  - 富文本内容需 sanitize

### 令牌存储
- **决策**: localStorage
- **理由**: 
  - 简单易用
  - 内部系统，XSS 风险可控
  - 支持跨标签页共享

### 敏感信息
- **决策**: 不在前端存储敏感信息
- **要求**:
  - 密码不明文显示
  - 不在 URL 传递敏感参数
  - 不在 console 输出敏感数据

---

## 6. API 通信

### HTTP 客户端
- **决策**: Axios
- **说明**: 成熟的 HTTP 客户端库，支持拦截器、请求取消等功能

### 国际化
- **决策**: i18next
- **支持语言**: 中文（简体）、英文
- **说明**: 功能强大的国际化解决方案

### 超时配置
- **决策**: 统一 10 秒超时
- **理由**: 
  - 内部系统网络环境稳定
  - 后端 API 响应时间目标 P95 ≤ 2 秒
  - 10 秒足够覆盖异常情况

### 错误处理
- **策略**: 统一拦截器处理
- **处理逻辑**:
  - 401: 跳转登录页
  - 403: 显示无权限提示
  - 500: 显示服务器错误提示
  - 网络错误: 显示网络异常提示

---

## 7. 图片处理

### 懒加载
- **决策**: 使用 Intersection Observer API
- **实现**: 
  - 图片进入视口前显示占位图
  - 进入视口后加载真实图片

### 占位图
- **决策**: 统一占位图
- **场景**:
  - 加载中: 灰色背景 + loading 动画
  - 加载失败: 默认"图片不可用"图标

### 图片优化
- **决策**: 后端不做处理，前端懒加载
- **理由**: MVP 阶段简化实现，后续可扩展

---

## 8. 响应式设计

### 设备支持
- **决策**: 仅桌面端
- **最小分辨率**: 1280 × 720
- **推荐分辨率**: 1920 × 1080
- **理由**: 内部系统，员工使用公司电脑

### 布局策略
- **员工端**: 固定宽度居中布局（最大宽度 1440px）
- **管理端**: 侧边栏 + 内容区弹性布局

---

## 9. 离线与 PWA

### 离线支持
- **决策**: 不支持
- **理由**: 内部系统，始终在线使用

### PWA
- **决策**: 不实现
- **理由**: MVP 阶段不需要

---

## 10. 错误监控

### 前端监控
- **决策**: 不集成第三方监控
- **理由**: MVP 阶段，依赖用户反馈

### 错误捕获
- **决策**: React ErrorBoundary + console.error
- **实现**:
  - 使用 React 的 ErrorBoundary 组件捕获渲染错误
  - 全局 window.onerror 捕获未处理异常
  - 错误信息输出到 console
